# 需求: 重构知识库目录结构

- **优先级**: 高
- **状态**: 待评估
- **描述**:
  当前知识库的内部结构较为扁平，不利于未来功能的扩展和管理。为了更好地支持实体、元数据、备份、回收站和编辑日志等功能，需要对知识库的目录结构进行重构。

- **新的目录结构**:
  ```
  library 目录
  ├── entities    # 存放所有的内容实体
  ├── meta.md     # 存放元数据文件
  ├── backups     # 存放备份文件
  ├── trash       # 作为删除后文件的回收站
  └── journeys    # 作为编辑日志（当前尚未实现）
  ```

- **影响范围**:
  - `GraphManager` 类及其相关工具需要更新以适应新的文件路径。
  - 现有的文件操作工具（如 `read_file`, `write_file` 等）需要调整其内部逻辑以正确处理新的目录结构。
  - 现有的 E2E 测试需要更新以反映新的文件路径和操作。

- **验收标准**:
  - 知识库能够按照新的目录结构正确创建和初始化。
  - 所有的实体文件能够正确地存储在 `entities` 目录下。
  - `meta.md` 文件能够正确地存储和读取。
  - 备份文件能够正确地存储在 `backups` 目录下。
  - 删除的文件能够正确地移动到 `trash` 目录下。
  - 现有的工具（如 `addEntities`, `readEntities` 等）能够在新结构下正常工作。
