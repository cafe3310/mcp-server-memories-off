# 需求: 细化并简化所有工具的返回格式

- **状态**: 待办
- **优先级**: 高

## 1. 目标

对 `refactored-design.md` 中定义的所有对外工具集（`manual`, `relation`, `entity`, `search` 系列）进行一次全面的设计审查和优化，确保每个工具都具备清晰、详细且对 LLM 友好的返回格式。

## 2. 核心原则

1.  **明确具体**: 每个工具的文档都必须明确描述其返回数据的结构，就像我们为 `search` 系列所做的那样。
2.  **LLM 友好**: 避免使用深度嵌套的 JSON/YAML 结构。优先选择扁平的数据格式，例如：
    -   简单的成功/失败状态消息。
    -   字符串列表。
    -   由简单键值对构成的对象列表。
3.  **信息适度**: 返回的信息应该恰到好处，既能让 LLM 理解操作结果，又不会因过于冗长而浪费上下文窗口。

## 3. 任务范围

-   [ ] **`manual` 系列**: 为 `read`, `update`, `add`, `delete` 操作定义简洁的返回格式。
-   [ ] **`relation` 系列**: 为 `create`, `delete`, `find` 操作定义清晰的返回结果。
-   [ ] **`entity` 系列**: 为所有的 CRUD、读取和维护操作（如 `rename`, `merge`）设计信息丰富的、扁平化的返回格式。
-   [ ] **`search` 系列**: 审查现有设计，确保其符合最新的“简化格式”原则。
-   [ ] **更新主设计文档**: 将所有细化后的设计更新到 `docs/refactored-design.md` 中。
