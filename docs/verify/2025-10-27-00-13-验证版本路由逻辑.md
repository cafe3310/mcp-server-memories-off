# 验证用例: 版本路由逻辑

- **目的**: 验证服务器能否根据环境变量 `MEM_VERSION` 正确启动 v1 或 v2 版本的服务。
- **前置条件**: 项目已成功构建 (`bun run build`)。

---

### 用例 1: 默认启动 v1 服务

1.  **执行操作**:
    ```bash
    bun ./dist/index.js
    ```
2.  **预期结果**:
    -   终端输出应包含 `Booting v1 server...`。

---

### 用例 2: 指定启动 v2 服务

1.  **执行操作**:
    ```bash
    MEM_VERSION=2 bun ./dist/index.js
    ```
2.  **预期结果**:
    -   终端输出应包含 `Booting v2 server...`。
